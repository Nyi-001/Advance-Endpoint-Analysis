# MBR and GTP analysis



## Introduction

**Disk** ဆိုတာ Data တွေကို `0` နဲ့ `1` Binary ပုံစံနဲ့ သိမ်းဆည်းပေးတဲ့ Storage Device ဖြစ်ပါတယ်။

**Partition** ဆိုတာ Physical Disk တစ်လုံးကို Logical အပိုင်းများအဖြစ် ခွဲထားခြင်းဖြစ်ပါတယ်။



### MBR (Master Boot Record)

**MBR** ဆိုတာ အဟောင်း Partitioning Scheme တစ်ခုဖြစ်ပြီး Disk ရဲ့ ပထမဆုံး Sector ဖြစ်တဲ့ **Sector 0** မှာ တည်ရှိပါတယ်။MBR ထဲမှာ—

- Boot Code
- Partition Table
- Boot Signature

တို့ ပါဝင်ပါတယ်။

MBR ရဲ့ အဓိကတာဝန်က Disk Partition တွေရဲ့ တည်နေရာကို သိမ်းဆည်းခြင်းနဲ့ Boot Process ကို စတင်ကူညီပေးခြင်းဖြစ်ပါတယ်။

ပုံမှန် 512-byte MBR ဖွဲ့စည်းပုံမှာ—

```

446 bytes → Boot Code
64 bytes  → Partition Table
2 bytes   → Boot Signature
```

MBR ရဲ့ အဓိက ကန့်သတ်ချက်များ

- Disk အရွယ်အစား 2 TB ဝန်းကျင်အထိသာ ထောက်ပံ့နိုင်တယ်။
- Primary Partition ၄ ခုသာ တိုက်ရိုက်ဖန်တီးနိုင်တယ်။
- Partition Table Backup မရှိဘူး။
- ပျက်စီးသွားရင် System Boot မတက်နိုင်ဘူး။
- Legacy BIOS နဲ့ အများဆုံးအသုံးပြုတယ်။

Primary Partition ၄ ခုထက်ပိုလိုရင် Partition တစ်ခုကို **Extended Partition** အဖြစ် ဖန်တီးပြီး အထဲမှာ Logical Partition တွေ ထပ်ခွဲရပါတယ်။



### GPT (GUID Partition Table)

**GPT** ဆိုတာ MBR ထက်ပိုမိုခေတ်မီပြီး လုံခြုံမှုနဲ့ ယုံကြည်စိတ်ချရမှု ပိုကောင်းတဲ့ Partitioning Scheme ဖြစ်ပါတယ်။

- ကြီးမားတဲ့ Disk များကို ထောက်ပံ့နိုင်တယ်။
- Partition အများအပြား ပြုလုပ်နိုင်တယ်။
- Backup Partition Table ပါရှိတယ်။
- Data ပျက်စီးမှုကို CRC Checksum နဲ့ စစ်ဆေးနိုင်တယ်။

GPT Disk ရဲ့ အခြေခံဖွဲ့စည်းပုံမှ

```
Sector 0       → Protective MBR
Sector 1       → Primary GPT Header
Next sectors   → Partition Entries
Disk အဆုံးပိုင်း → Backup GPT Entries နှင့် Backup GPT Header
```

GPT ရဲ့ အားသာချက်များ

- 2 TB ထက်ကြီးတဲ့ Disk တွေကို ထောက်ပံ့နိုင်တယ်။
- Partition အများအပြား ဖန်တီးနိုင်တယ်။
- Extended Partition မလိုဘူး။
- Backup GPT ရှိတယ်။
- CRC Checksum အသုံးပြုတယ်။
- Partition တစ်ခုချင်းစီမှာ GUID ရှိတယ်။
- UEFI System နဲ့ အသုံးများတယ်။

Windows မှာ GPT Disk တစ်လုံးကို ပုံမှန်အားဖြင့် Partition **128 ခုအထိ** အသုံးပြုနိုင်ပါတယ်။





**Sector** ဆိုတာ Disk ပေါ်မှာ Data သိမ်းဆည်းဖို့ အသုံးပြုတဲ့ အသေးဆုံး Fixed-size Storage Unit တစ်ခုဖြစ်ပါတယ်။ Regular Sector size is **512 bytes** and **4096 bytes**.

**Bootloader** ဆိုတာ Operating System ကို Memory ထဲသို့ Load လုပ်ပေးတဲ့ Program ဖြစ်ပါတယ်။

- Windows Boot Manager

- GRUB

- systemd-boot

**BIOS — Basic Input/Output System** ဆိုတာ Computer ဖွင့်တဲ့အခါ Hardware တွေကို စစ်ဆေးပြီး Boot Device ကို ရှာပေးတဲ့ အဟောင်း Firmware System ဖြစ်ပါတယ်။

BIOS ကို အများအားဖြင့် MBR နဲ့ တွဲဖက်အသုံးပြုပါတယ်။

**EFI System Partition** ဆိုတာ UEFI System မှာ Bootloader `.efi` ဖိုင်တွေ သိမ်းထားတဲ့ Partition ဖြစ်ပါတယ်။

အများအားဖြင့် FAT32 File System ကို အသုံးပြုပါတယ်။

**Active Partition** ဆိုတာ MBR System မှာ Bootloader ပါရှိတယ်လို့ သတ်မှတ်ထားတဲ့ Partition ဖြစ်ပါတယ်။

**GUID — Globally Unique Identifier** ဆိုတာ GPT Disk နဲ့ Partition တစ်ခုချင်းစီကို ထူးခြားစွာ သတ်မှတ်ပေးတဲ့ Identifier ဖြစ်ပါတယ်။

**Protective MBR** ဆိုတာ GPT Disk ရဲ့ Sector 0 မှာရှိတဲ့ အထူး MBR Entry ဖြစ်ပါတယ်။

အဟောင်း MBR-only Tools တွေက GPT Disk ကို Empty Disk လို့ မှားယွင်းစွာ မယူဆစေဖို့ ကာကွယ်ပေးပါတယ်။

**CRC Checksum** ဆိုတာ GPT Header သို့မဟုတ် Partition Table ပျက်စီး၊ ပြောင်းလဲမှုရှိ၊ မရှိ စစ်ဆေးဖို့ အသုံးပြုတဲ့ Error-detection Value ဖြစ်ပါတယ်။

**Bootkit** ဆိုတာ Computer ရဲ့ Boot Process ကို တိုက်ခိုက်တဲ့ Malware အမျိုးအစားဖြစ်ပါတယ်။

- MBR ကို ပြောင်းလဲနိုင်တယ်။
- Bootloader ကို ကူးစက်နိုင်တယ်။
- Operating System မတက်ခင် အလုပ်လုပ်နိုင်တယ်။
- Antivirus မစတင်မီ Malware ကို Run နိုင်တယ်။
- System ကို Boot မတက်အောင် လုပ်နိုင်တယ်။



### The Difference between MBR and GPT

| အချက်                  | MBR                | GPT                      |
| ---------------------- | ------------------ | ------------------------ |
| အပြည့်အစုံ                | Master Boot Record | GUID Partition Table     |
| အသုံးများသော Firmware    | BIOS               | UEFI                     |
| Disk အရွယ်အစား         | 2 TB ဝန်းကျင်      | အလွန်ကြီးမားသော Disk များ |
| Partition အရေအတွက်     | Primary ၄ ခု        | Windows တွင် ပုံမှန် 128 ခု |
| Backup Partition Table | မရှိ                | ရှိ                       |
| Integrity Checking     | မရှိ                | CRC Checksum ရှိ          |
| Partition Identifier   | Type Code          | GUID                     |
| ယုံကြည်စိတ်ချရမှု          | နည်း               | ပိုကောင်း                  |
| ခေတ်အခြေအနေ            | အဟောင်း            | ခေတ်မီ                    |

## Boot Process

**Boot Process** ဆိုတာ ကွန်ပျူတာကို Power ဖွင့်ချိန်မှ Operating System အသုံးပြုနိုင်တဲ့အချိန်အထိ လုပ်ဆောင်သွားတဲ့ အဆင့်များဖြစ်ပါတယ်။

![](/images/6645aa8c024f7893371eb7ac-1737110666165.svg)



### Power-On the System

Boot process ရဲ့ ပထမဆုံးအဆင့်ကတော့ Power ခလုတ်ကို နှိပ်လိုက်တာနဲ့ စတင်ပါတယ်။ အဲဒီအခါ Motherboard ဆီကို လျှပ်စစ်လှိုင်းတွေ ပို့လွှတ်လိုက်ပြီး Hardware အစိတ်အပိုင်းအားလုံးကို စတင်အလုပ်လုပ်စေပါတယ်။

![](/images/6645aa8c024f7893371eb7ac-1737110666101.svg)

- **CPU** ဟာ လျှပ်စစ်လှိုင်းတွေကို ပထမဆုံးလက်ခံရရှိတဲ့ အစိတ်အပိုင်းဖြစ်ပြီး ရှေ့ဆက်အလုပ်လုပ်ဖို့ ညွှန်ကြားချက်အချို့ လိုအပ်ပါတယ်။

- CPU က အဲဒီညွှန်ကြားချက်တွေကို Motherboard ပေါ်မှာ တပ်ဆင်ထားတဲ့ Chipset တစ်ခုဆီကနေ ရယူပြီး အလုပ်လုပ်ပါတယ်။ အဲဒီ Chipset ကို **BIOS** သို့မဟုတ် **UEFI** လို့ခေါ်ပြီး၊ ၎င်းအထဲမှာ Boot process ကို ဘယ်လိုစတင်ရမယ်ဆိုတဲ့ ညွှန်ကြားချက်တွေ ပါဝင်ပါတယ်။



##### BIOS and UEFI

BIOS (Basic Input/Output System) နဲ့ UEFI (Unified Extensible Firmware Interface) တို့ဟာ Hardware အစိတ်အပိုင်းအားလုံး မှန်မှန်ကန်ကန် အလုပ်လုပ်ခြင်း ရှိ/မရှိ စစ်ဆေးပေးဖို့ တာဝန်ရှိပါတယ်။ ကွန်ပျူတာတစ်လုံးဟာ BIOS ဒါမှမဟုတ် UEFI Firmware တစ်ခုခုကို အသုံးပြုပါတယ်။

- **BIOS:** BIOS ကို ဆယ်စုနှစ်ပေါင်းများစွာ အသုံးပြုခဲ့ကြပြီး တချို့သော Hardware တွေမှာ အခုထိ သုံးနေတုန်းပါပဲ။ သူက အခြေခံ 16-bit mode နဲ့ပဲ အလုပ်လုပ်ပြီး Storage ပမာဏ 2 Terabytes (TB) အထိပဲ ထောက်ပံ့ပေးနိုင်ပါတယ်။ အရေးအကြီးဆုံး မှတ်သားထားရမယ့် အချက်ကတော့ BIOS ဟာ **MBR (Master Boot Record)** Partitioning စနစ်ကိုသာ ထောက်ပံ့ပေးတာ ဖြစ်ပါတယ်။

- **UEFI:** UEFI ကတော့ BIOS နေရာမှာ အစားထိုးဝင်ရောက်လာတာဖြစ်ပြီး 32-bit နဲ့ 64-bit mode တွေနဲ့ အလုပ်လုပ်နိုင်ကာ Storage ပမာဏ 9 Zettabytes (ZB) အထိတောင် ထောက်ပံ့ပေးနိုင်ပါတယ်။ UEFI မှာ System Boot နေစဉ်အတွင်း လုံခြုံရေးအတွက် **Secure Boot** လုပ်ဆောင်ချက် ပါဝင်ပါတယ်။ ဒါ့အပြင် Boot Code ပျက်စီးသွားခဲ့ရင်တောင် Backup ကနေ ပြန်လည်ရယူနိုင်တဲ့ စနစ် (Redundancy) လည်း ပါဝင်ပါတယ်။ BIOS နဲ့ မတူတဲ့အချက်ကတော့ UEFI ဟာ **GPT (GUID Partition Table)** Partitioning စနစ်ကို အသုံးပြုခြင်းပဲ ဖြစ်ပါတယ်။



#### How to check if your system uses BIOS or UEFI firmware

window os:

![](/images/6645aa8c024f7893371eb7ac-1731683551194.png)

၁။ Keyboard မှ `Windows Key + R` ကို တွဲနှိပ်ပြီး Run dialog box ကို ဖွင့်ပါ။ ၂။ အဲဒီအထဲမှာ `msinfo32` လို့ ရိုက်ထည့်ပြီး Enter ခေါက်ပါ။ ၃။ ဒါဆိုရင် System Summary (စနစ်အချက်အလက် အကျဉ်းချုပ်) ပြတင်းပေါက် ပွင့်လာပါမယ်။ ၄။ အဲဒီအထဲက **BIOS Mode** ဆိုတဲ့ အကွက်မှာ

- **Legacy** လို့ ပြနေရင် သင်သည် **BIOS** ကို အသုံးပြုထားတာဖြစ်ပြီး၊
- **UEFI** လို့ ပြနေရင်တော့ **UEFI** ကို အသုံးပြုထားတာ ဖြစ်ပါတယ်။



### Power-On-Self-Test (POST)

စက်ကို စတင်ဖွင့်လိုက်ပြီဖြစ်ပြီး CPU ဟာ BIOS/UEFI က ညွှန်ကြားချက်တွေကို စတင်လုပ်ဆောင်နေပြီ ဖြစ်ပါတယ်။ အဲဒီနောက်မှာ BIOS သို့မဟုတ် UEFI က စနစ်ရဲ့ Hardware အစိတ်အပိုင်းတွေ အားလုံး ကောင်းမွန်စွာ အလုပ်လုပ်ခြင်း ရှိ/မရှိ သေချာစေဖို့ **Power-On-Self-Test (POST)** ဆိုတဲ့ စစ်ဆေးမှုကို စတင်လုပ်ဆောင်ပါတယ်။

![](/images/6645aa8c024f7893371eb7ac-1737110666092.svg)

ဒီလိုစစ်ဆေးနေချိန်မှာ သင့်စက်ဆီကနေ Beep (ဘီပီ) အသံ တစ်ချက် ဒါမှမဟုတ် အချက်များစွာ မြည်လာတာကို ကြားရနိုင်ပါတယ်။ ဒါဟာ BIOS/UEFI က Hardware တွေမှာ ချို့ယွင်းချက် (Error) တစ်ခုခုရှိနေရင် သင့်ကို အသိပေးဆက်သွယ်တဲ့ နည်းလမ်းဖြစ်ပြီး၊ ဥပမာ - "Keyboard not found" (ကီးဘုတ် ရှာမတွေ့ပါ) စတဲ့ Error မက်ဆေ့ချ်တွေကို မျက်နှာပြင်ပေါ်မှာ ပြသပေးလေ့ရှိပါတယ်။



### Locate the Bootable Device

BIOS/UEFI က POST စစ်ဆေးမှုကို အောင်မြင်စွာ လုပ်ဆောင်ပြီးတဲ့အခါ၊ Operating System (OS) သွင်းထားတဲ့ SSD, HDD ဒါမှမဟုတ် USB စတဲ့ Boot တက်နိုင်တဲ့ Device (**Bootable Device**) တွေကို ရှာဖွေရမယ့် အချိန်ရောက်ပါပြီ။

![](/images/6645aa8c024f7893371eb7ac-1737110666094.svg)

Bootable Device ကို ရှာတွေ့သွားပြီဆိုတာနဲ့ အဲဒီ Device ထဲက အချက်အလက်တွေကို စတင်ဖတ်ပါတော့တယ်။ ဒီနေရာမှာ **MBR** နဲ့ **GPT** တို့ရဲ့ အခန်းကဏ္ဍ စတင်လာပါပြီ။ အဆိုပါ Storage Device ရဲ့ ပထမဆုံး Sector (အစပိုင်းနေရာ) မှာ MBR (Master Boot Record) ဒါမှမဟုတ် GPT (GUID Partition Table) တစ်ခုခု ပါဝင်နေမှာ ဖြစ်ပါတယ်။ ဒီအဆင့်ကစပြီး MBR သို့မဟုတ် GPT ဟာ Boot process တစ်ခုလုံးကို ဆက်လက်ထိန်းချုပ်သွားမှာ ဖြစ်ပါတယ်။

MBR ဟာ Disk ရဲ့ ပထမဆုံး Sector မှာ နေရာယူထားတဲ့အတွက် အရွယ်အစား **512 bytes** အတိအကျ ရှိပါတယ်။ ပထမဆုံးလိုင်းကနေ စတင်တယ်ဆိုတာ သိပေမယ့် MBR Code တွေ ဘယ်နေရာမှာ ဆုံးသွားသလဲဆိုတာကို ဘယ်လိုသိနိုင်မလဲ။ အဖြေက ရှင်းပါတယ်။

- Hexadecimal (Hex) စနစ်မှာ ဂဏန်းနှစ်လုံးတွဲ (ဥပမာ- `4A`, `B2`) တိုင်းဟာ **1 byte** ကို ကိုယ်စားပြုပါတယ်။
- ကျွန်တော်တို့ သုံးနေတဲ့ Hex Editor ရဲ့ အတန်း (Row) တစ်တန်းစီမှာ Hex ဂဏန်း ၁၆ တွဲ (16 bytes) ပါဝင်ပါတယ်။
- ဒါကြောင့် **16 bytes x 32 rows = 512 bytes** ဖြစ်တဲ့အတွက် ပထမဆုံး အတန်း ၃၂ တန်းဟာ MBR တစ်ခုလုံးဖြစ်တယ်လို့ အလွယ်တကူ သိနိုင်ပါတယ်။
- MBR အဆုံးသတ်ကို သိနိုင်တဲ့ နောက်ထပ်နည်းလမ်းတစ်ခုကတော့ **MBR Signature** ကို ရှာဖွေခြင်းပါ။ MBR Code ရဲ့ အဆုံးသတ်ကို အမြဲတမ်း `55 AA` ဆိုတဲ့ Hex ဂဏန်းလေးတွေနဲ့ အဆုံးသတ်လေ့ရှိပါတယ်။ Hex Editor ထဲမှာ အဲဒီ `55 AA` ကို ရှာတွေ့ရင် အဲဒီနေရာဟာ MBR ရဲ့ အဆုံးသတ် (၅၁၁ နှင့် ၅၁၂ မြောက် ဘိုက်များ) ဖြစ်ပါတယ်။

Forensic (မှုခင်းစုံစမ်းစစ်ဆေးရေး) လုပ်တဲ့အခါ ကျွန်တော်တို့ အဓိက အာရုံစိုက်လေ့လာရမှာက ဒုတိယအကွက်ဖြစ်တဲ့ **Hexadecimal Bytes** တွေပဲ ဖြစ်ပါတယ်။

### Bootloader Code (Bytes 0-445)

MBR ရဲ့ ပထမဆုံး အစိတ်အပိုင်းကတော့ Bootloader Code ဖြစ်ပါတယ်။ စုစုပေါင်း 512 bytes ရှိတဲ့အထဲမှာ သူက ၄၄၆ ဘိုက် (446 bytes) အထိ နေရာယူထားပါတယ်။

- ဒီ Bootloader Code ထဲမှာ **Initial Bootloader** (ကနဦး Bootloader) ပါဝင်ပါတယ်။
- Initial Bootloader ဟာ MBR ထဲမှာ ပထမဆုံး စတင်အလုပ်လုပ်တဲ့ ကုတ် (Code) ဖြစ်ပါတယ်။
- သူ့ရဲ့ အဓိကတာဝန်ကတော့ MBR ထဲမှာရှိတဲ့ Partition Table ဆီကနေတစ်ဆင့် OS သွင်းထားတဲ့ **Bootable Partition (Boot တက်နိုင်သော အခန်း)** ကို ရှာဖွေပေးဖို့ပဲ ဖြစ်ပါတယ်။



### Partitions Table (Bytes 446-509)

MBR ရဲ့ ဒုတိယမြောက် အစိတ်အပိုင်းကတော့ **Partition Table** ဖြစ်ပြီး၊ ၆၄ ဘိုက် (64 bytes) အရွယ်အစား ရှိပါတယ်။ ဒီ Table ထဲမှာ Disk ပေါ်မှာ ခွဲခြားထားတဲ့ Partition တွေ အားလုံးရဲ့ အချက်အလက်တွေ ပါဝင်ပါတယ်။

![](/images/Screenshot_2026-07-16_21-21-25.png)

၁။ Disk ထဲက Partition တစ်ခုမှာ ကွန်ပျူတာ Boot တက်ဖို့ လိုအပ်တဲ့ Operating System (OS) ဖိုင်တွေ ပါဝင်ပါတယ်။ အဲဒါကို **Bootable Partition** လို့ ခေါ်ပါတယ်။ 

၂။ MBR က Bootloader Code ကနေ စတင်အလုပ်လုပ်တဲ့ Initial Bootloader ဟာ ဒီ Partition Table ထဲကို ဝင်ရှာပြီး Bootable Partition ကို ရှာဖွေပါတယ်။ 

၃။ ရှာတွေ့သွားရင် အဲဒီ Bootable Partition ထဲကနေ **Second Bootloader** (ဒုတိယ Bootloader) ကို ထပ်မံဆွဲတင် (Load) ပေးပါတယ်။ 

၄။ အဲဒီ Second Bootloader ကမှတစ်ဆင့် Operating System ရဲ့ အသက်သွေးကြောဖြစ်တဲ့ **Kernel** ကို ဆက်လက် ဆွဲတင်ပေးသွားတာ ဖြစ်ပါတယ်။



![](/images/6645aa8c024f7893371eb7ac-1733741844707.png)

Bootloader Code တွေက ရှုပ်ထွေးပေမယ့်၊ ဒီ Partition Table ထဲက Hexadecimal ဂဏန်းတွေကတော့ တိကျတဲ့ အချက်အလက်တွေကို တိုက်ရိုက်ပြောပြနေပါတယ်။ Partition Table ဓာတ်ပုံထဲက ပထမဆုံး Partition ရဲ့ (၁၆) ဘိုက်ကို နမူနာယူပြီး ဒီ Hex ဂဏန်းတွေရဲ့ အဓိပ္ပာယ်ကို ခွဲခြမ်းစိတ်ဖြာကြည့်ရအောင်။

![](/images/6645aa8c024f7893371eb7ac-1731927561284.png)

| **ဘိုက် တည်နေရာ (Position)** | **ဘိုက် အရေအတွက် (Length)** | **ဘိုက် တန်ဖိုးများ (Hex Bytes)** | **အစိတ်အပိုင်း အမည် (Field Name)** |
| -------------------------- | ------------------------- | ----------------------------- | ------------------------------- |
| `0`                        | 1 byte                    | `80`                          | **Boot Indicator**              |
| `1-3`                      | 3 bytes                   | `20 21 00`                    | **Starting CHS Address**        |
| `4`                        | 1 byte                    | `07`                          | **Partition Type**              |
| `5-7`                      | 3 bytes                   | `FE FF FF`                    | **Ending CHS Address**          |
| `8-11`                     | 4 bytes                   | `00 08 00 00`                 | **Starting LBA Address**        |
| `12-15`                    | 4 bytes                   | `00 B0 23 03`                 | **Number of Sectors**           |

- **Boot Indicator (Boot တက်နိုင်မှု အခြေအနေ):** ဒီ ၁ ဘိုက်တည်းသော ဂဏန်းက ဒီ Partition ဟာ ကွန်ပျူတာ Boot တက်လို့ရတဲ့ (OS ပါတဲ့) အခန်းလား၊ မဟုတ်ဘူးလားဆိုတာကို ပြပါတယ်။ တန်ဖိုးအနေနဲ့ `80` နဲ့ `00` နှစ်မျိုးတည်းသာ ရှိပါတယ်။ `80` ဆိုရင် Bootable (Boot တက်နိုင်သည်) ဖြစ်ပြီး၊ `00` ဆိုရင် Not bootable (Boot မတက်နိုင်ပါ) လို့ အဓိပ္ပာယ်ရပါတယ်။ Windows စနစ်တွေမှာဆိုရင် `C:` Drive ဟာ များသောအားဖြင့် Bootable ဖြစ်တဲ့အတွက် သူ့ရဲ့ Boot Indicator က `80` ဖြစ်နေမှာပါ။

- **Starting CHS Address:** **Cylinder Head Sector (CHS)** ဆိုတာကတော့ ဒီ Partition ဟာ Disk ပေါ်က ဘယ် Physical (ရုပ်ပိုင်းဆိုင်ရာ) နေရာမှာ စတင်သလဲဆိုတာကို ပြတဲ့ ဘိုက် ၃ ခု ဖြစ်ပါတယ်။ ဒါပေမယ့် နောက်ပိုင်းမှာ ပိုပြီးရိုးရှင်းလွယ်ကူတဲ့ လော့ဂျစ် (Logical) လိပ်စာဖြစ်တဲ့ LBA (Logical Block Addressing) ကို ပြောင်းသုံးလာကြတဲ့အတွက် ဒီ CHS Address က သိပ်အရေးမကြီးတော့ပါဘူး။

- **Partition Type (Partition အမျိုးအစား):** Partition တိုင်းမှာ NTFS, FAT32 စတဲ့ သီးခြား Filesystem (ဖိုင်စနစ်) တွေ ရှိကြပါတယ်။ ဒီ ၁ ဘိုက်က အဆိုပါ Filesystem အမျိုးအစားကို ကိုယ်စားပြုပါတယ်။ ကျွန်တော်တို့ နမူနာယူထားတဲ့ Partition မှာ `07` လို့ ပြနေတဲ့အတွက် သူသည် **NTFS Partition** ဖြစ်တယ်လို့ သိနိုင်ပါတယ်။ Filesystem တစ်ခုချင်းစီမှာ ကိုယ်ပိုင် သီးသန့် Byte ကုဒ်တွေ ရှိကြပါတယ်။

- **Ending CHS Address:** ဒါကတော့ Partition ဆုံးသွားတဲ့ နေရာရဲ့ Physical (CHS) လိပ်စာကို ပြတဲ့ ဘိုက် ၃ ခုပါ။ အစောကပြောခဲ့သလိုပဲ LBA ကိုသာ အဓိက သုံးတဲ့အတွက် ဒါကလည်း အရေးပါမှု သိပ်မရှိလှပါဘူး။

- **Starting LBA Address:** **Logical Block Addressing (LBA)** ဆိုတာကတော့ Partition စတင်တဲ့နေရာကို ပြတဲ့ Logical လိပ်စာ ၄ ဘိုက် ဖြစ်ပါတယ်။ CHS က ရုပ်ပိုင်းဆိုင်ရာ လိပ်စာဖြစ်လို့ ရှာဖွေရ ခက်ခဲပေမယ့်၊ ဒီ LBA Address ကတော့ Hex Editor ထဲမှာ Partition စတဲ့နေရာကို အလွယ်တကူ ရှာဖွေနိုင်အောင် ကူညီပေးပါတယ်။ Forensic သဘောတရားအရ ပြောရရင် ဒီ LBA လိပ်စာကို သုံးပြီး Disk တစ်ခုထဲက ဖျောက်ထားတဲ့ (Hidden) ဒါမှမဟုတ် ဖျက်ပစ်လိုက်တဲ့ (Deleted) Partition တွေဆီကနေ Data တွေကို ပြန်လည်ရှာဖွေ ဖော်ထုတ် (Data Carving) နိုင်ပါတယ်။ နမူနာပုံမှာ `00 08 00 00` ဆိုတဲ့ LBA ကိုတွေ့ရမှာဖြစ်ပြီး နောက်သင်ခန်းစာမှာ ဒီလိပ်စာကိုသုံးပြီး Partition ကို ဘယ်လိုရှာမလဲဆိုတာ ဆက်ကြည့်ပါမယ်။

- **Number of Sectors (Sector အရေအတွက်):** နောက်ဆုံး ၄ ဘိုက်ကတော့ ဒီ Partition ထဲမှာ Sector ပေါင်း ဘယ်လောက် ပါဝင်သလဲ ဆိုတာကို ပြပါတယ်။ ဒီဂဏန်းကို အသုံးပြုပြီး Partition ရဲ့ စုစုပေါင်း အရွယ်အစား (Size) ကို ကျွန်တော်တို့ တွက်ချက်နိုင်ပါတယ်။



### Locating the Partition

#### Step 1 (Reversing Little-Endian)

ဒီ LBA ဘိုက်တွေကို ကွန်ပျူတာထဲမှာ **Little-endian** ပုံစံနဲ့ သိမ်းဆည်းထားတာပါ။ Little-endian ဆိုတာ အငယ်ဆုံးတန်ဖိုးရှိတဲ့ဘိုက် (Least Significant Byte - LSB) ကို အရင်ဆုံးထားပြီး၊ အကြီးဆုံးတန်ဖိုးရှိတဲ့ဘိုက် (Most Significant Byte - MSB) ကို နောက်ဆုံးမှထားတဲ့ စနစ်ဖြစ်ပါတယ်။

![](/images/Screenshot_2026-07-16_21-53-15.png)

ဒါကြောင့် ပထမဆုံးအနေနဲ့ ဒီဘိုက်တွေကို မှန်ကန်တဲ့အစီအစဉ်ဖြစ်အောင် ပြောင်းပြန် (Reverse) လှန်ပေးရပါမယ်။

- မူလတန်ဖိုး: `00 08 00 00`
- ပြောင်းပြန်လှန်ပြီးတန်ဖိုး: **`00 00 08 00`**



#### Step 2 (Changing to Decimal)

နောက်တစ်ဆင့်ကတော့ ဒီ Hex တန်ဖိုး (`00 00 08 00`) ကို သာမန်ကျွန်တော်တို့သုံးတဲ့ Decimal (ဒဿမဂဏန်းစနစ်) အဖြစ် ပြောင်းလဲရပါမယ်။

![](/images/Screenshot_2026-07-16_21-58-40.png)

ဒီနမူနာအရ Decimal တန်ဖိုး **`2048`** ဆိုပြီး ရရှိပါမယ်။ (ဆိုလိုသည်မှာ ဤ Partition သည် ၂၀၄၈ ခုမြောက် Sector တွင် စတင်သည်)။



#### Step 3 (Calculating the byte offset)

Sector အရေအတွက် (2048) ကို ရပြီဆိုရင်၊ Sector တစ်ခုရဲ့ စံသတ်မှတ်ထားတဲ့ အရွယ်အစားဖြစ်တဲ့ **512 bytes** နဲ့ မြှောက်ပေးရပါမယ်။

- **2048 x 512 = 1,048,576**

ဒီ `1,048,576` ဆိုတဲ့ တန်ဖိုးဟာ Disk ပေါ်မှာ Partition စတင်တည်ရှိနေတဲ့ အတိအကျနေရာ (Exact Byte Location/Offset) ပဲ ဖြစ်ပါတယ်။

## What if GPT

နည်းပညာတွေ တိုးတက်လာတာနဲ့အမျှ ယခင်အသုံးပြုခဲ့တဲ့ **BIOS + MBR** အတွဲအစပ်ဟာ ခေတ်မမီတော့ဘဲ ကန့်သတ်ချက်တွေ ရှိလာတာကြောင့် ခေတ်သစ်စနစ်တွေမှာ **UEFI + GPT** အတွဲအစပ်နဲ့ အစားထိုးခဲ့ကြပါတယ်။ GPT ဟာ UEFI Firmware သုံးတဲ့ စနစ်တွေနဲ့ ကောင်းမွန်စွာ ကိုက်ညီမှုရှိပြီး လက်ရှိ ကွန်ပျူတာအများစုမှာ အဓိက သုံးစွဲနေကြတဲ့ Partition စနစ် ဖြစ်ပါတယ်။

### Structure of GPT

ကျွန်တော်တို့ ယခင်သင်ခန်းစာမှာ လေ့လာခဲ့သလို **MBR** ဟာ Disk ရဲ့ ပထမဆုံး Sector (512 bytes) တစ်ခုတည်းမှာပဲ တည်ရှိပြီး အလုပ်လုပ်သွားတာ ဖြစ်ပါတယ်။ဒါပေမယ့် **GPT** ကတော့ အဲဒီလို မဟုတ်ပါဘူး။ GPT ရဲ့ အချက်အလက်တွေနဲ့ လုပ်ဆောင်ချက်တွေကို Disk ရဲ့ Sector အမြောက်အမြားပေါ်မှာ ပျံ့နှံ့သိမ်းဆည်းထားပြီး အဓိက အစိတ်အပိုင်း **(၅) ခု** နဲ့ ဖွဲ့စည်းထားပါတယ်။	

![](/images/6645aa8c024f7893371eb7ac-1737110838398.svg)



### Protective MBR

ပုံမှန်အားဖြင့် GPT partition စနစ်သုံးထားတဲ့ Disk တစ်ခုကို ကိုင်တွယ်ဖို့ UEFI firmware လိုအပ်ပါတယ်။ ဒါပေမယ့် စနစ်ဟောင်း (Legacy System) အချို့ဟာ Disk က GPT ဖြစ်နေပေမယ့် BIOS firmware ကို သုံးနေတုန်းပါပဲ။ BIOS က MBR နဲ့ပဲ အလုပ်လုပ်ဖို့ ဒီဇိုင်းဆွဲထားပြီး၊ UEFI ကတော့ GPT နဲ့ အလုပ်လုပ်ဖို့ ဒီဇိုင်းဆွဲထားတာကြောင့် ဒါဟာ ပြဿနာတစ်ခု ဖြစ်လာနိုင်ပါတယ်။

ဒီပြဿနာကို ဖြေရှင်းဖို့ GPT မှာ **Protective MBR** (ကာကွယ်ရေး MBR) ဆိုတာ ပါဝင်လာပါတယ်။ Protective MBR ဟာ GPT စနစ်သုံး Disk ရဲ့ ပထမဦးဆုံး Sector (Sector 0) မှာ တည်ရှိပါတယ်။

> **သူ့ရဲ့ အဓိက တာဝန်ကဘာလဲ?** BIOS စနစ်ဟောင်းတွေနဲ့ ရောထွေးပြီး ဒေတာတွေ ပျက်စီးမသွားစေဖို့ ဖြစ်ပါတယ်။ သာမန် BIOS စနစ်သုံး စက်တွေက ဒီ Disk ကို မြင်တဲ့အခါ Partition မရှိဘူးအထင်နဲ့ အသစ်ပြန်ခွဲတာမျိုး၊ သို့မဟုတ် ဒေတာတွေကို ကျော်ရေး (Overwrite) တာမျိုး မလုပ်မိစေဖို့ **"ဒီ Disk က GPT သုံးထားတာမို့လို့ ဝင်မရှုပ်ပါနဲ့"** လို့ သတိပေးတဲ့ ဆိုင်းဘုတ်တစ်ခုအနေနဲ့ အလုပ်လုပ်ပေးတာ ဖြစ်ပါတယ်။

Here are the three portion of protective MBR

![](/images/6645aa8c024f7893371eb7ac-1734943004432.png)

1. ##### Bootloader code (from 0 to 445 bytes)

ပုံမှန် MBR က ကုဒ်တွေနဲ့ မတူပါဘူး။ သူက Boot တက်နေစဉ်မှာ ဘာအလုပ်မှ မလုပ်ပါဘူး။

ပုံမှန် MBR bootloader ရှိနေသယောင် ဟန်ပြလုပ်ထားတာ သက်သက်ပဲ ဖြစ်ပါတယ်။

အခြေအနေအများစုမှာတော့ ဒီနေရာမှာ `00` တွေချည်းပဲ ရှိနေတတ်ပြီး၊ တစ်ခါတစ်ရံမှာတော့ စနစ်ဟောင်းတွေနဲ့ ကိုက်ညီမှုရှိစေဖို့ (Legacy Compatibility) Placeholder ကုတ်အချို့ ပါဝင်နေတတ်ပါတယ်။

2. ##### Partition table (from 446 to 509 bytes)

ဒီ Table မှာ Partition တစ်ခုတည်း (ပထမဆုံး ၁၆ ဘိုက်) သာ အသက်ဝင်ပြီး ၎င်းရဲ့ တာဝန်ကတော့ စနစ်ကို EFI Partition ဆီ လမ်းကြောင်းလွှဲ (Redirect) ပေးဖို့ပဲ ဖြစ်ပါတယ်။

ကျန်တဲ့ Partition ၃ ခုနေရာမှာတော့ `0` (သုည) တွေပဲ ဖြည့်ထားပါတယ်။

ဒီတစ်ဦးတည်းသော Partition ထဲမှာ အရေးအကြီးဆုံးအချက်ကတော့ **၄ ခုမြောက် ဘိုက် (4th byte)** ဖြစ်ပြီး၊ ၎င်းဘိုက်တန်ဖိုးကို **`EE`** လို့ သတ်မှတ်ထားပါတယ်။ ဒါဟာ ဒီစက်က GPT စနစ်သုံးထားတဲ့ Disk ဖြစ်တယ်ဆိုတာကို သက်သေပြတဲ့ သင်္ကေတ ဖြစ်ပါတယ်။

3. MBR signature (from 510 to 511)

ပုံမှန် MBR အတိုင်းပါပဲ၊ **`55 AA`** ဖြစ်ပြီး Protective MBR ရဲ့ အဆုံးသတ်ကို သတ်မှတ်ပေးပါတယ်။



```ပထမဆုံး Sector (Sector 0) ရဲ့ Partition Table မှာ Partition Type ဘိုက်နေရာမှာ **`EE`** ကို တွေ့လိုက်ရပြီဆိုရင် ဒါဟာ MBR စနစ်အစစ်မဟုတ်ဘဲ **GPT စနစ်သုံးထားတဲ့ Protective MBR ဖြစ်တယ်** ဆိုတာကို ချက်ချင်း သိရှိနိုင်မှာ ဖြစ်ပါတယ်။```





### Primary GPT Header

Protective MBR တည်ရှိရာ Sector 0 (`55 AA` ဖြင့် ပြီးဆုံးသည့်နေရာ) ပြီးနောက် ကပ်လျက်ရှိသော နောက်တစ်ဘိုက် (Sector 1 ၏ အစမှတ်) မှစတင်ပြီး **Primary GPT Header** တည်ရှိသည်။  Disk ပေါ်ရှိ Partition များအားလုံးကို စနစ်တကျစီမံရန် **အသေးစိတ်မြေပုံ (Blueprint)** တစ်ခုအဖြစ် ဆောင်ရွက်ပေးသည်။

GPT Header တစ်ခုလုံးသည် Sector တစ်ခုစာဖြစ်သော **$512 \text{ bytes}$** ပမာဏရှိသော်လည်း၊ တကယ်တမ်းအချက်အလက် သိမ်းဆည်းရန်အတွက် ပထမဆုံး **$92 \text{ bytes}$** ကိုသာ အသုံးပြုပါသည်။ ကျန်ရှိသော အရွယ်အစားပြည့်မီစေရန် ကျန် $420 \text{ bytes}$ နေရာလွတ်များကို `00` (Null bytes) များဖြင့်သာ ဖြည့်သွင်း (Padding) ထားလေ့ရှိသည်။ ထို့ကြောင့် Forensic စုံစမ်းစစ်ဆေးရာတွင် ကျွန်ုပ်တို့သည် အဓိပ္ပာယ်ရှိသော အချက်အလက်များပါဝင်သည့် ပထမဆုံး $92 \text{ bytes}$ ကိုသာ အဓိက အာရုံစိုက်လေ့လာရန် လိုအပ်ပါသည်

![](/images/6645aa8c024f7893371eb7ac-1732091427207.png)

| ----------------- | --------- | ---------------------------- | ---------------------------- |
| **0-7**           | 8 bytes   | `45 46 49 20 50 41 52 54`    | Signature                    |
| **8-11**          | 4 bytes   | `00 00 01 00`                | Revision                     |
| **12-15**         | 4 bytes   | `5C 00 00 00`                | Header Size                  |
| **16-19**         | 4 bytes   | `71 89 13 1C`                | CRC32 of Header              |
| **20-23**         | 4 bytes   | `00 00 00 00`                | Reserved                     |
| **24-31**         | 8 bytes   | `01 00 00 00 00 00 00 00`    | Current LBA                  |
| **32-39**         | 8 bytes   | `AF 32 CF 1D 00 00 00 00`    | Backup LBA                   |
| **40-47**         | 8 bytes   | `22 00 00 00 00 00 00 00`    | First Usable LBA             |
| **48-55**         | 8 bytes   | `8E 32 CF 1D 00 00 00 00`    | Last Usable LBA              |
| **56-71**         | 16 bytes  | `1D F1 B0 D6 43 BE 37 4E...` | Disk GUID                    |
| **72-79**         | 8 bytes   | `02 00 00 00 00 00 00 00`    | Partition Entry Array LBA    |
| **80-83**         | 4 bytes   | `80 00 00 00`                | Number of Partition Entries  |
| **84-87**         | 4 bytes   | `80 00 00 00`                | Size of Each Partition Entry |
| **88-91**         | 4 bytes   | `41 0D C0 22`                | CRC32 of Partition Array     |

**Signature:** အမြဲတမ်း `45 46 49 20 50 41 52 54` ဖြစ်ပြီး ASCII စာသားအားဖြင့် **"EFI PART"** ဟု အဓိပ္ပာယ်ရပါတယ်။ ၎င်းသည် မှန်ကန်သော GPT header တစ်ခုဖြစ်ကြောင်း သက်သေပြပါတယ်။

**Revision:** GPT ဗားရှင်းကို ပြပါတယ်။ အများအားဖြင့် `00 00 01 00` (Version 1.0) ဖြစ်ပါတယ်။

**Header Size:** GPT header ၏ စုစုပေါင်း အရွယ်အစားဖြစ်ပါတယ်။ ဥပမာပြထားသော `5C 00 00 00` ကို ပြောင်းပြန်လှန်လိုက်လျှင် `00 00 00 5C` ဖြစ်ပြီး ဆယ်လီစိတ် (decimal) အားဖြင့် **92 bytes** ဖြစ်ပါတယ်။

**CRC32 of Header:** Header ၏ အချက်အလက်များ မှန်ကန်မှုရှိမရှိ (ပျက်စီး/ပြုပြင်ခံရခြင်း ရှိမရှိ) စစ်ဆေးပေးသော checksum ဖြစ်ပါတယ်။

**CRC32 of Partition Array:** Partition စာရင်း (array) တစ်ခုလုံး၏ မှန်ကန်မှုကို စစ်ဆေးပေးသော checksum ဖြစ်ပါတယ်။

**Disk GUID:** Disk တစ်ခုနှင့်တစ်ခု မတူညီစေရန် သတ်မှတ်ပေးထားသော 16-byte Globally Unique Identifier (သီးသန့်အမှတ်အသား) ဖြစ်ပါတယ်။

**Reserved:** နောင်လာမည့် နည်းပညာအသစ်များတွင် အသုံးပြုနိုင်ရန် ချန်လှပ်ထားသော နေရာဖြစ်ပြီး (`00 00 00 00`) သုညများသာ ပါဝင်ပါတယ်။

**Current LBA:** ယခု GPT header တည်ရှိနေသော နေရာကို ပြပါတယ်။ `01 00 00 00 00 00 00 00` ကို ပြောင်းပြန်လှန်ပါက `1` ဖြစ်သည့်အတွက် (Sector 1) ကို ဆိုလိုပါတယ်။

**Backup LBA:** Disk ၏ အဆုံးပိုင်းတွင် အမြဲသိမ်းဆည်းထားလေ့ရှိသော အရန် (Backup) GPT header တည်ရှိမည့် နေရာကို ညွှန်ပြပါတယ်။

**First Usable LBA:** Partition data များကို စတင်ရေးသားနိုင်သည့် ပထမဆုံး နေရာ (Sector) ဖြစ်ပါတယ်။

**Last Usable LBA:** Partition များ ရေးသားနိုင်သည့် နောက်ဆုံး နေရာ ဖြစ်ပါတယ်။ ယင်းနောက်ပိုင်းတွင် Backup GPT မှလွဲ၍ မည်သည့်ဒေတာမှ ရေးသားခွင့် မရှိပါ။

**Partition Entry Array LBA:** Partition စာရင်းများ စတင်မည့် နေရာဖြစ်ပါတယ်။ `02 00...` သည် ယခု header အပြီး (Sector 2) မှ ချက်ချင်းစတင်မည်ဟု ဆိုလိုပါတယ်။

**Number of Partition Entries:** Disk တွင် အများဆုံး ထည့်သွင်းနိုင်သော Partition အရေအတွက် ဖြစ်ပါတယ်။ `80 00 00 00` ကို ပြောင်းပြန်လှန်ပါက hex အားဖြင့် `80` ဖြစ်ပြီး ဆယ်လီစိတ်အားဖြင့် Partition **128** ခု ဖြစ်ပါတယ်။ (MBR ၏ အများဆုံး 4 ခုသာ ရသည့်စနစ်ထက် အလွန်ပိုမိုကောင်းမွန်လာခြင်း ဖြစ်ပါတယ်)။

**Size of Each Partition Entry:** Partition စာရင်း တစ်ခုချင်းစီအတွက် နေရာမည်မျှ ယူမည်ကို ပြပါတယ်။ `80 00 00 00` သည် တစ်ခုလျှင် **128 bytes** ယူမည်ဟု ဆိုလိုပါတယ်။



### Partition Entry Array

Sector 0 နေရာတွင် Protective MBR ရှိပြီး၊ Sector 1 နေရာကို GPT header က ယူထားကြောင်း ကျွန်ုပ်တို့ တွေ့ခဲ့ရပြီး ဖြစ်ပါတယ်။ ယခု Sector 2 မှစတင်၍ **Partition Entry Array** စတင်လာပြီ ဖြစ်ပါတယ်။ ၎င်းသည် MBR တွင်ပါဝင်သော partition table နှင့် အလုပ်လုပ်ပုံ ဆင်တူသော်လည်း ကွဲပြားချက်အချို့ ရှိပါတယ်။GPT disk တစ်ခုတွင် စုစုပေါင်း partition ၁၂၈ ခု အထိ ဖန်တီးနိုင်ပြီး၊ ဤ Partition Entry Array တွင် အဆိုပါ partition အားလုံး၏ အချက်အလက်များကို သိမ်းဆည်းထားပါတယ်။

Partition တစ်ခုချင်းစီအတွက် နေရာ **128 bytes** ယူပါတယ်။

ဥပမာအားဖြင့် သင့် disk တွင် partition ၆ ခုသာ အလုပ်လုပ်နေပါက၊ ထို ၆ ခုအတွက်သာ data များကို 128-byte block များအနေဖြင့် တွေ့ရမည်ဖြစ်ပြီး၊ ကျန်ရှိနေသော partition အလွတ် ၁၂၂ ခုလုံးကို သုည (`00`) များဖြင့်သာ ပြသထားမည် ဖြစ်ပါတယ်။

#### Partition Type GUID

Partition ၏ အမျိုးအစား (ဥပမာ - EFI System Partition, Basic Data Partition စသည်) ကို ဖော်ပြသော GUID ဖြစ်ပါတယ်။ ဤနေရာရှိ 16 bytes များကို **mixed endian** (little-endian နှင့် big-endian ရောနှောလျက်) ပုံစံဖြင့် သိမ်းဆည်းထားပါတယ်။ ထို့ကြောင့် standard GUID ပုံစံသို့ ပြောင်းလဲလိုပါက အောက်ပါအဆင့်များအတိုင်း ပြုလုပ်ရပါမယ်-

1. **ပထမ 4 bytes:** `28 73 2A C1` ကို little-endian ဖြစ်၍ ပြောင်းပြန်လှန်ပါ (`C1 2A 73 28`)။
2. **နောက် 2 bytes:** `1F F8` ကို little-endian ဖြစ်၍ ပြောင်းပြန်လှန်ပါ (`F8 1F`)။
3. **နောက် 2 bytes:** `D2 11` ကို little-endian ဖြစ်၍ ပြောင်းပြန်လှန်ပါ (`11 D2`)။
4. **နောက် 2 bytes:** `BA 4B` သည် big-endian ဖြစ်၍ အတိုင်းထားပါ (`BA 4B`)။
5. **နောက်ဆုံး 6 bytes:** `00 A0 C9 3E C9 3B` သည် big-endian ဖြစ်၍ အတိုင်းထားပါ (`00 A0 C9 3E C9 3B`)။

ရရှိလာမည့် GUID မှာ `C12A7328-F81F-11D2-BA4B-00A0C93EC93B` ဖြစ်ပါတယ်။ ဤ GUID များကို အင်တာနက်တွင် ရှာဖွေကြည့်ပါက မည်သည့် partition အမျိုးအစားဖြစ်ကြောင်း သိရှိနိုင်ပါတယ်။ 



#### EFI system partitions

ESP သည် GPT disk တိုင်းတွင် မပါမဖြစ်ပါဝင်ရသော အလွန်အရေးကြီးသည့် boot ဆိုင်ရာ အစိတ်အပိုင်း ဖြစ်ပါတယ်။ MBR စနစ်တွင် bootloaders များကို bootcode နေရာ၌ သိမ်းဆည်းသော်လည်း၊ GPT စနစ်တွင်တော့ `.efi` extension ဖြင့် ဖိုင်များခွဲ၍ ဤ ESP ထဲတွင် သိမ်းဆည်းပါတယ်။ Bootloader က ESP ထဲမှ အလုပ်စတင်ပြီးနောက်၊ OS ၏ kernel ကို ဆက်လက်ခေါ်ယူ (load) လုပ်ပါတယ်။ ထိုမှတစ်ဆင့် drivers များ၊ services များ အလုပ်လုပ်လာပြီး သုံးစွဲသူထံသို့ User Interface ရောက်ရှိလာခြင်း ဖြစ်ပါတယ်။

1. ##### Unique Partition GUID

Disk ပေါ်ရှိ partition များကို တစ်ခုနှင့်တစ်ခု မတူညီစေရန် သီးခြားခွဲခြားပေးသော သီးသန့် GUID ဖြစ်ပါတယ်။ ၎င်းကိုလည်း Partition Type GUID ကဲ့သို့ပင် mixed endian ပုံစံဖြင့် တွက်ချက်ဖတ်ရှုရပါမယ်။

2. ##### Starting LBA

ဤ Partition ကို Disk ပေါ်တွင် စတင်ရေးသားထားသော (စတင်သည့်) LBA လိပ်စာ နေရာဖြစ်ပါတယ်။

3. ##### Ending LBA

ဤ Partition ၏ ဧရိယာ ဆုံးသတ်သည့် LBA လိပ်စာ နေရာဖြစ်ပါတယ်

4. ##### Attributes

Partition နှင့် ပတ်သက်သော လုပ်ဆောင်ချက်များ (Flags) ပါဝင်ပါတယ်။ ဥပမာ - ဤ partition သည် boot တက်၍ရသော (bootable) partition လား၊ ဖျောက် (hidden) ထားသော partition လား စသည်ဖြင့် သတ်မှတ်ပေးပါတယ်။

5. ##### Partition Name

Partition ၏ အမည်ကို စာသား (String) ပုံစံဖြင့် သိမ်းဆည်းထားပြီး အရွယ်အစားအားဖြင့် 72 bytes ယူထားပါတယ်။ စာသားများကို **UTF-16 encoding** ဖြင့် သိမ်းဆည်းထားခြင်းဖြစ်ပြီး၊ အွန်လိုင်းရှိ hex-to-string decoder များအသုံးပြု၍ အလွယ်တကူ စာသားအဖြစ် ပြန်လည်ပြောင်းလဲ ဖတ်ရှုနိုင်ပါတယ်။



### Backup GPT Header

MBR ကို GPT နဲ့ အစားထိုးရတဲ့ အကြီးမားဆုံး အကြောင်းရင်းတစ်ခုကတော့ **Redundancy (အရန်အသင့် ရှိနေခြင်း)** ကြောင့် ဖြစ်ပါတယ်။

MBR စနစ်မှာဆိုရင် hardware ပြဿနာတွေကြောင့်ဖြစ်စေ၊ malicious attacks (ဗိုင်းရပ်စ် သို့မဟုတ် ဆော့ဖ်ဝဲလ်ဆိုးများ တိုက်ခိုက်မှု) ကြောင့်ဖြစ်စေ MBR code တွေ ပျက်စီး (corrupt) သွားရင် recovery ပြန်လုပ်ဖို့ အခွင့်အလမ်း အလွန်နည်းပါတယ်။ ဒါပေမဲ့ GPT မှာတော့ အဓိက အစိတ်အပိုင်းအားလုံးအတွက် Backup (အရန်သိမ်းဆည်းမှု) စနစ်တွေ ပါရှိတဲ့အတွက် ဒေတာတွေ ပျက်စီးဆုံးရှုံးမှုကနေ စိတ်ချလက်ချ ကာကွယ်ပေးနိုင်ပါတယ်။

**တည်နေရာ:** Disk ရဲ့ အဆုံးသတ် နောက်ဆုံး sector (last sector of the disk) မှာ တည်ရှိပါတယ်။ (ပုံထဲက `last LBA n` နေရာ ဖြစ်ပါတယ်)။

**လုပ်ဆောင်ချက်:** အကယ်၍ ရှေ့ဆုံးက Primary GPT Header သာ ပျက်စီးသွားခဲ့ရင် ဒီ Backup header ကို အသုံးပြုပြီး recovery ပြန်လုပ်နိုင်ပါတယ်။ ၎င်းထဲမှာ Primary GPT Header နဲ့ ထပ်တူညီတဲ့ အချက်အလက်တွေပဲ ပါဝင်ပါတယ်။



### Backup Partition Entry Array

**တည်နေရာ:** Disk ရဲ့ အဆုံးပိုင်း၊ Backup GPT Header ရဲ့ ချက်ချင်းအရှေ့ကပ်လျက်နေရာမှာ သိမ်းဆည်းထားပါတယ်။ (ပုံထဲက `last LBA n-32` နေရာ ဖြစ်ပါတယ်)။

**လုပ်ဆောင်ချက်:** မူရင်း Partition Entry Array တစ်ခုခု ချို့ယွင်းပျက်စီးသွားခဲ့ရင် အရန်သင့် အသုံးပြုနိုင်မယ့် ဘေးကင်းလုံခြုံရေးစနစ် (fail-safe mechanism) အဖြစ် အလုပ်လုပ်ပါတယ်။ ၎င်းမှာလည်း မူရင်း Partition Entry Array ထဲက အချက်အလက်တွေအတိုင်း ထပ်တူညီစွာ သိမ်းဆည်းထားတာ ဖြစ်ပါတယ်။
